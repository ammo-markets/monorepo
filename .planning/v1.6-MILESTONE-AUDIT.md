## Tech Debt Summary

### Phase 28: Data Flow Completion
1. **ActivityLog updatedAt alias** — The `updatedAt` field in the activity API response is `row.createdAt.toISOString()`. The ActivityLog model has no `@updatedAt` column. Sorting by "latest state change" is functionally identical to sorting by insertion time. Acceptable for finalized-order activity logs but not a true state-change sort.
2. **Orphaned ShippingAddress route** — `POST /api/redeem/shipping` has full implementation (auth, ownership verification, state validation, upsert) but is never called by the redeem UI flow. The redeem flow saves the address to the user's profile (`PATCH /api/users/profile`) instead. No `ShippingAddress` record is linked to individual order records, meaning the admin cannot see per-order delivery addresses.

### Phase 29: Security Hardening
3. **KYC GET error swallowing** — `use-kyc.ts` catch block returns `NONE` status on 500 errors from the KYC GET query. Pre-existing by design as an auth fallback. Does not affect the mutation hook (SEC-02).

### Phase 30: Architecture & Contract Hardening
4. **Admin stats hardcoded calibers** — `/api/admin/stats/route.ts` line 9 hardcodes `["9MM", "556", "22LR", "308"]` instead of deriving from `CONTRACT_ADDRESSES.fuji.calibers`. Adding a new caliber to shared config will not auto-surface in admin stats.
5. **Market route slice indices** — `/api/market/route.ts` uses `.slice(0, 4)` / `.slice(4)` to split oracle vs supply multicall results. Will silently break if caliber count changes from 4.
6. **Human verification pending** — Foundry test suite (72 tests) and Fuji deployment liveness need human confirmation via `forge test` and live RPC query.

### Total: 6 items across 3 phases
