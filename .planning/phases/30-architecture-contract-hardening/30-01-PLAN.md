<task type="auto">
  <name>Task 1: Add deadline validation and price sanity bounds to CaliberMarket.sol</name>
  <files>packages/contracts/src/CaliberMarket.sol</files>
  <action>
Add two new custom errors to CaliberMarket.sol:
- `error DeadlineInPast();` — for CNTR-01
- `error PriceTooLow();` — for CNTR-02 lower bound
- `error PriceTooHigh();` — for CNTR-02 upper bound

Add a configurable price sanity bound state variable:
- `uint256 public maxPriceDeviationBps = 5000;` — defaults to 50% (keeper can finalize within +/-50% of oracle price)
- Add owner-only setter: `function setMaxPriceDeviation(uint256 bps) external onlyOwner { if (bps == 0 || bps > 10_000) revert InvalidBps(); maxPriceDeviationBps = bps; }`

**CNTR-01 — Deadline validation in startMint:**
After `if (maxSlippageBps > 10_000) revert InvalidBps();`, add:
```solidity
if (deadline != 0 && deadline <= uint64(block.timestamp)) revert DeadlineInPast();
```
This allows deadline=0 (no deadline) but rejects deadlines that have already passed.

**CNTR-01 — Deadline validation in startRedeem:**
After `if (tokenAmount == 0) revert InvalidAmount();`, add:
```solidity
if (deadline != 0 && deadline <= uint64(block.timestamp)) revert DeadlineInPast();
```

**CNTR-02 — Price sanity bounds in finalizeMint:**
After the existing `if (actualPriceX18 == 0) revert InvalidPrice();` check in finalizeMint, add:
```solidity
uint256 oraclePrice = oracle.getPrice();
if (oraclePrice > 0) {
    uint256 floor = (oraclePrice * (10_000 - maxPriceDeviationBps)) / 10_000;
    uint256 ceiling = (oraclePrice * (10_000 + maxPriceDeviationBps)) / 10_000;
    if (actualPriceX18 < floor) revert PriceTooLow();
    if (actualPriceX18 > ceiling) revert PriceTooHigh();
}
```
This checks the keeper-supplied price against the current oracle price at finalization time. If the oracle returns 0 (broken), the sanity check is skipped (the existing `if (actualPriceX18 == 0)` already guards against zero-price finalization). The `maxPriceDeviationBps` is configurable by the owner.

Note: Do NOT change any existing error types or remove any existing checks. Only add new validation logic.
