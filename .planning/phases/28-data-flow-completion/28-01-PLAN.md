**Stats API (`apps/web/app/api/stats/route.ts`):**
1. Replace `select: { amount: true }` with `select: { usdcAmount: true }` in the mintOrders query
2. Replace `Number(BigInt(order.amount)) / 1e6` with string-safe computation:
   - Sum usdcAmount values as BigInt, then format as string: `(sumBigInt / BigInt(1e6)).toString()` or use a string division. The key is the final JSON value must be a string, not a Number.
   - For totalVolumeUsd: accumulate BigInt sum of all usdcAmount values (skip nulls), divide by 1_000_000n, convert to string. Return as `totalVolumeUsd: string` in response.
3. For roundsTokenized: accumulate BigInt sum of totalMinted values, divide by 10n**18n, convert to string. Return as `roundsTokenized: string` in response.
4. For the `stats` array items: `totalMinted`, `totalRedeemed`, `netSupply` are already Prisma String fields, so they're already safe. No change needed there.

**Market API (`apps/web/app/api/market/route.ts`):**

1. Change `totalSupply: Math.floor(Number(formatUnits(supply, 18)))` to `totalSupply: formatUnits(supply, 18)` (returns string like "1000.0"). Or better: compute as BigInt division `(supply / 10n**18n).toString()` for a clean integer string. Use `formatUnits` but return the string directly.
2. Keep `pricePerRound: Number(priceX18) / 1e18` as-is since prices are small numbers that fit in Number safely (well under 2^53).
3. `priceX18` is already `.toString()` -- safe.

**Admin Stats API (`apps/web/app/api/admin/stats/route.ts`):**

1. Change `totalSupply: Math.floor(Number(formatUnits(supplies[i]!, 18)))` to `totalSupply: formatUnits(supplies[i]!, 18)` or `(supplies[i]! / 10n**18n).toString()`.
2. `treasuryUsdc` uses `formatUnits(usdcBalance, 6)` which already returns string -- safe, no change needed.
