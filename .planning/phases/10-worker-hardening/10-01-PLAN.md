Output: Two new handler files and an updated indexer that fetches and processes all 9 event types.
</objective>

  <action>
Create `apps/worker/src/handlers/refund.ts` following the exact same pattern as `mint.ts` and `redeem.ts`:

1. Define `MintRefundedArgs` interface: `{ orderId: bigint; user: \`0x${string}\`; refundAmount: bigint; reasonCode: number }` (matches the Solidity event signature in ICaliberMarket.sol line 61).

2. Define `RedeemCanceledArgs` interface: `{ orderId: bigint; user: \`0x${string}\`; unlockedTokens: bigint; reasonCode: number }` (matches ICaliberMarket.sol line 64).

3. Implement `handleMintRefunded(tx: PrismaTx, args: MintRefundedArgs, meta: EventMeta): Promise<void>`:
   - Resolve caliber from `meta.address` using `addressToCaliber`
   - Convert to Prisma caliber using `CALIBER_TO_PRISMA`
   - Use `tx.order.updateMany` to find the matching order: `{ onChainOrderId: args.orderId.toString(), caliber: prismaCaliber, type: "MINT", status: "PENDING" }` and update to `{ status: "FAILED" }`
   - Log: `[refund] MintRefunded orderId=${args.orderId} reason=${args.reasonCode}`

4. Implement `handleRedeemCanceled(tx: PrismaTx, args: RedeemCanceledArgs, meta: EventMeta): Promise<void>`:
   - Same pattern as above but match `type: "REDEEM"` and update to `status: "CANCELLED"`
   - Log: `[refund] RedeemCanceled orderId=${args.orderId} reason=${args.reasonCode}`

Import types from the same places as existing handlers: `PrismaTx` from `../lib/cursor`, `EventMeta` and `addressToCaliber` from `../lib/constants`, `CALIBER_TO_PRISMA` from `@ammo-exchange/shared`, `Caliber as PrismaCaliber` from `@ammo-exchange/db`.

Export all types and functions.

**Part A: Create `apps/worker/src/handlers/lifecycle.ts`**

These are log-only handlers (no DB writes) for operational visibility:

1. `handlePaused(meta: EventMeta, args: { by: \`0x${string}\` }): void` -- logs `[lifecycle] Market PAUSED by ${args.by} at block ${meta.blockNumber} (contract: ${meta.address})`

2. `handleUnpaused(meta: EventMeta, args: { by: \`0x${string}\` }): void` -- logs `[lifecycle] Market UNPAUSED by ${args.by} at block ${meta.blockNumber} (contract: ${meta.address})`

3. `handleMintFeeUpdated(meta: EventMeta, args: { oldBps: bigint; newBps: bigint }): void` -- logs `[lifecycle] MintFee updated ${args.oldBps} -> ${args.newBps} bps (contract: ${meta.address})`

4. `handleRedeemFeeUpdated(meta: EventMeta, args: { oldBps: bigint; newBps: bigint }): void` -- logs `[lifecycle] RedeemFee updated ${args.oldBps} -> ${args.newBps} bps (contract: ${meta.address})`

5. `handleMinMintUpdated(meta: EventMeta, args: { oldMin: bigint; newMin: bigint }): void` -- logs `[lifecycle] MinMint updated ${args.oldMin} -> ${args.newMin} (contract: ${meta.address})`

These do NOT need `PrismaTx` -- they are pure logging functions. They DO take `EventMeta` for context.

4. Update the `totalEvents` count in `pollOnce()` to include all 9 event arrays.

Keep the existing block-order sorting logic (blockNumber then logIndex) -- it naturally handles the new events.
