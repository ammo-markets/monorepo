**1. `apps/web/hooks/use-redeem-transaction.ts`** -- Two-step approve AmmoToken + startRedeem hook.

Add `"use client"` directive.

Import: `useWriteContract`, `useWaitForTransactionReceipt` from "wagmi"; `parseUnits` from "viem"; `CaliberMarketAbi`, `AmmoTokenAbi` from "@ammo-exchange/contracts/abis"; `CONTRACT_ADDRESSES` from "@ammo-exchange/shared"; `type { Caliber }` from "@ammo-exchange/shared".

Export `useRedeemTransaction(caliber: Caliber)`:

- Derive `marketAddress = CONTRACT_ADDRESSES.fuji.calibers[caliber].market`
- Derive `tokenAddress = CONTRACT_ADDRESSES.fuji.calibers[caliber].token`
- Create TWO separate `useWriteContract` instances (same pattern as useMintTransaction):
  - Approve instance for AmmoToken: `{ data: approveHash, error: approveError, isPending: isApprovePending, writeContract: writeApprove, reset: resetApprove }`
  - Redeem instance: `{ data: redeemHash, error: redeemError, isPending: isRedeemPending, writeContract: writeRedeem, reset: resetRedeem }`
- Create TWO `useWaitForTransactionReceipt` instances:
  - `{ isLoading: isApproveConfirming, isSuccess: isApproveConfirmed } = useWaitForTransactionReceipt({ hash: approveHash })`
  - `{ isLoading: isRedeemConfirming, isSuccess: isRedeemConfirmed } = useWaitForTransactionReceipt({ hash: redeemHash })`

Methods:

- `approve(tokenAmount: string)` -- calls `writeApprove({ address: tokenAddress, abi: AmmoTokenAbi, functionName: "approve", args: [marketAddress, parseUnits(tokenAmount, 18)] })`. NOTE: approve targets the AmmoToken contract address (NOT CaliberMarket). The spender (args[1]) is the market address. AmmoToken has 18 decimals.
- `startRedeem(tokenAmount: string, deadline: bigint)` -- calls `writeRedeem({ address: marketAddress, abi: CaliberMarketAbi, functionName: "startRedeem", args: [parseUnits(tokenAmount, 18), deadline] })`. NOTE: tokenAmount is in rounds (user-facing) but must be converted to 18-decimal token units.
- `reset()` -- calls both `resetApprove()` and `resetRedeem()`

Return all state + methods: `{ approve, approveHash, approveError, isApprovePending, isApproveConfirming, isApproveConfirmed, startRedeem, redeemHash, redeemError, isRedeemPending, isRedeemConfirming, isRedeemConfirmed, reset }`.

**2. `apps/web/app/api/users/kyc/route.ts`** -- GET/POST KYC status with auto-approve.

Import: `type { NextRequest }` from "next/server", `z` from "zod", `prisma` from "@ammo-exchange/db".

Define `walletSchema = z.object({ wallet: z.string().regex(/^0x[a-fA-F0-9]{40}$/) })`.

**GET handler** `/api/users/kyc?wallet=0x...`:

1. Read `wallet` from searchParams
2. Validate with walletSchema, return 400 on failure
3. Query `prisma.user.findUnique({ where: { walletAddress: parsed.data.wallet.toLowerCase() }, select: { kycStatus: true } })`
4. Return `{ kycStatus: user?.kycStatus ?? "NONE" }`

**POST handler** `/api/users/kyc` with body `{ wallet: "0x..." }`:

1. Parse JSON body, return 400 on invalid JSON
2. Validate with walletSchema, return 400 on failure
3. Upsert user: `prisma.user.upsert({ where: { walletAddress: wallet.toLowerCase() }, create: { walletAddress: wallet.toLowerCase(), kycStatus: "APPROVED" }, update: { kycStatus: "APPROVED" } })`
4. Return `{ kycStatus: user.kycStatus }`

This auto-approves for testnet. In production, this would integrate a real KYC provider.

Follow existing API route conventions from Phase 3: zod validation, lowercase wallet addresses, Response.json pattern.
