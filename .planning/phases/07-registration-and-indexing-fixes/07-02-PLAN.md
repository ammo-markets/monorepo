    **Step 2: Fix pollOnce in `apps/worker/src/indexer.ts`**

    Add `DEPLOYMENT_BLOCK` to the imports from `./lib/constants`:

    ```typescript
    import {
      BATCH_SIZE,
      MARKET_ADDRESSES,
      CURSOR_KEY,
      DEPLOYMENT_BLOCK,
    } from "./lib/constants";
    ```

    In the `pollOnce` function, replace the `fromBlock` calculation (line ~164):

    Before:
    ```typescript
    const fromBlock = (cursor?.lastBlock ?? 0n) + 1n;
    ```

    After:
    ```typescript
    // Use deployment block as floor -- never scan before contracts existed
    const rawFrom = (cursor?.lastBlock ?? 0n) + 1n;
    const fromBlock = rawFrom < DEPLOYMENT_BLOCK ? DEPLOYMENT_BLOCK : rawFrom;
    ```

    This means: if cursor is null (0n + 1n = 1n) or below deployment block, jump straight to deployment block. If cursor is already past deployment block, use cursor as normal.

    **Step 3: Add progress logging during backfill**

    In the `while (batchStart <= currentBlock)` loop in pollOnce, add progress logging. After the `batchEnd` calculation and before `fetchEvents`, add:

    ```typescript
    const totalBlocks = Number(currentBlock - fromBlock + 1n);
    const scannedBlocks = Number(batchStart - fromBlock);
    const progressPct = totalBlocks > 0 ? Math.floor((scannedBlocks / totalBlocks) * 100) : 100;
    console.log(
      `[indexer] Scanning batch ${batchStart}..${batchEnd} (${progressPct}% complete, ${totalEventsProcessed} events found)`
    );
    ```

    Place this INSIDE the while loop, right after the `batchEnd` calculation. The `totalBlocks` and `scannedBlocks` calculations should be inside the loop so they use the current `batchStart`.

    Also update the final summary log at the end of pollOnce to include blocks scanned count:

    ```typescript

    );
    ```

    After:
    ```typescript
    const blocksScanned = Number(currentBlock - fromBlock + 1n);
    console.log(
    ```
