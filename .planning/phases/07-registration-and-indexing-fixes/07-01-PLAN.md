    Create a new API route at `apps/web/app/api/users/register/route.ts` that handles user auto-registration.

    The endpoint accepts POST with JSON body `{ wallet: "0x..." }`:
    1. Validate wallet address using zod: `z.string().regex(/^0x[a-fA-F0-9]{40}$/)`
    2. Normalize to lowercase: `parsed.data.wallet.toLowerCase()`
    3. Use `prisma.user.upsert` with:
       - `where: { walletAddress: normalizedWallet }`
       - `create: { walletAddress: normalizedWallet }` (kycStatus defaults to NONE via Prisma schema)
       - `update: {}` (no-op if already exists -- idempotent)
    4. Return `Response.json({ walletAddress: user.walletAddress, kycStatus: user.kycStatus })` with status 200

    Follow the exact same patterns as `apps/web/app/api/users/kyc/route.ts`:
    - Import `type { NextRequest }` from "next/server"
    - Import `{ z }` from "zod"
    - Import `{ prisma }` from "@ammo-exchange/db"
    - Return 400 with `{ error, details }` on validation failure

    The upsert makes this fully idempotent -- calling it multiple times for the same wallet is safe and returns the existing record.

  <action>
    Modify `apps/web/hooks/use-wallet.ts` to call the register endpoint when a wallet connects.

    Add a `useEffect` that fires when `account.address` changes to a defined value:

    ```typescript
    import { useEffect } from "react";
    ```

    Inside the `useWallet` function, after the existing hook calls and before the return statement, add:

    ```typescript
    // Auto-register user in database on wallet connect
    useEffect(() => {
      if (!account.address) return;

      fetch("/api/users/register", {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({ wallet: account.address }),
      }).catch(() => {
        // Silent fail -- registration is best-effort.
        // User record will be created by worker on first on-chain event as fallback.
      });
    }, [account.address]);
    ```

    Key design decisions:
    - Fire-and-forget: Do NOT await or block the UI. Registration is best-effort.
    - Silent catch: If it fails (network error, server down), the worker's connectOrCreate in event handlers serves as fallback.
    - Dependency on `account.address`: Fires on initial connect AND on account switch (MetaMask account change).
    - No loading/error state surfaced to user -- this is invisible infrastructure.
    - Do NOT add any state variables (useState) -- keep the hook's return signature unchanged.
  </action>
