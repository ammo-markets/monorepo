**Fix `as any` casts (QUAL-01):**

1. **`apps/web/lib/errors.ts` line 52** (`const cause = (error as any).cause`):
   Replace with proper typing. The `cause` property exists on `Error` in modern TS but viem/wagmi errors have nested cause structures. Use:

   ```ts
   const cause =
     (error as BaseError).walk?.() ??
     (error.cause as Record<string, unknown> | undefined);
   ```

   Or more precisely, since we want `cause.data.errorName`:

   ```ts
   interface ContractErrorCause {
     data?: { errorName?: string };
     reason?: string;
   }
   const cause: ContractErrorCause | undefined =
     "cause" in error ? (error.cause as ContractErrorCause) : undefined;
   ```

   Remove the `// eslint-disable-next-line` comment above it.

2. **`apps/web/hooks/use-token-balances.ts` line 42** (`contracts: contracts as any`):
   The issue is that wagmi's `useReadContracts` expects a specific readonly tuple type but the spread operator loses the tuple specificity. Fix by removing `as const` from the inner mapped array and instead typing the contracts array properly. The simplest fix: define the contracts array with explicit type annotation that matches wagmi's expected input, or use a const assertion on each individual contract object. Since this is a known wagmi typing issue, the pragmatic fix is:

   ```ts
   // Build contracts with proper typing
   const usdcContract = {
     address: fuji.usdc,
     abi: erc20Abi,
     functionName: "balanceOf" as const,
     args: address ? ([address] as const) : undefined,
   };

   const tokenContracts = CALIBERS.map((caliber) => ({
     address: fuji.calibers[caliber].token,
     abi: AmmoTokenAbi,
     functionName: "balanceOf" as const,
     args: address ? ([address] as const) : undefined,
   }));

   const allContracts = [usdcContract, ...tokenContracts];
   ```

   Then pass `contracts: allContracts` (wagmi should accept this). If it still complains, try:

   ```ts
   const { data, isLoading, refetch } = useReadContracts({
     contracts: allContracts.map((c) => ({ ...c })),
     query: { enabled: isConnected && !!address },
   });
   ```

   If wagmi's types are truly incompatible without `as any`, use a more specific cast: `as Parameters<typeof useReadContracts>[0]["contracts"]` instead of `as any`. But try the restructured approach first.
   Remove the `// eslint-disable-next-line` comment.
