  <action>
    1. Install recharts: `pnpm --filter @ammo-exchange/pitchdeck add recharts@2.15.4`

    2. Create `apps/pitchdeck/lib/useDeck.ts` -- custom hook managing slide navigation:
       - State: `currentSlide` (number, 0-indexed), `direction` ("left" | "right")
       - Takes `totalSlides: number` parameter
       - Navigation functions: `next()`, `prev()`, `goTo(index)`, `goFirst()`, `goLast()`
       - Boundary logic: `goTo` returns early if index < 0 or >= totalSlides (no wrap-around)
       - `direction` updates: going forward = "right", going backward = "left"
       - Keyboard handler in useEffect on `keydown`:
         - ArrowRight / Space -> next() + e.preventDefault()
         - ArrowLeft -> prev() + e.preventDefault()
         - Home -> goFirst() + e.preventDefault()
         - End -> goLast() + e.preventDefault()
       - Returns: { currentSlide, direction, totalSlides, next, prev, goTo, goFirst, goLast, isFirst, isLast }
       - Mark file with "use client" at top
       - Use useCallback for all navigation functions to stabilize useEffect dependencies
       - Follow the exact pattern from 25-RESEARCH.md Pattern 1

    3. Add CSS @keyframes to `apps/pitchdeck/app/globals.css` (append after existing content):
       - `@keyframes slideInRight` -- from { opacity: 0; transform: translateX(30px); } to { opacity: 1; transform: translateX(0); }
       - `@keyframes slideInLeft` -- from { opacity: 0; transform: translateX(-30px); } to { opacity: 1; transform: translateX(0); }
       - IMPORTANT: Use @keyframes animation (NOT CSS transition) because key prop remount has no "from" state for transitions
  </action>
  <verify>

    - `grep -r "recharts" apps/pitchdeck/package.json` shows recharts dependency
    - `grep "@keyframes slideIn" apps/pitchdeck/app/globals.css` finds both keyframe definitions
