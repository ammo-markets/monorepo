Replace hardcoded Tailwind zinc/amber classes with inline styles using CSS variables. The rest of the app uses `style={{ color: "var(--text-muted)" }}` patterns extensively, so follow the same convention.

**admin-sidebar.tsx mapping:**

- `border-zinc-800` -> `style={{ borderColor: "var(--border-default)" }}`
- `bg-zinc-950` -> `style={{ backgroundColor: "var(--bg-primary)" }}`
- `text-amber-400` -> `style={{ color: "var(--brass)" }}`
- `text-zinc-100` -> `style={{ color: "var(--text-primary)" }}`
- `text-zinc-400` -> `style={{ color: "var(--text-secondary)" }}`
- `text-zinc-200` -> `style={{ color: "var(--text-primary)" }}`
- `bg-amber-400/10` -> `style={{ backgroundColor: "var(--brass-muted)" }}`
- `hover:bg-zinc-800` -> use `onMouseEnter/onMouseLeave` with `var(--bg-tertiary)` OR convert to a Tailwind arbitrary class `hover:bg-[var(--bg-tertiary)]`. Prefer the Tailwind arbitrary class approach for hover states since it's cleaner.
- `hover:text-zinc-200` -> `hover:text-[var(--text-primary)]`

For the `isActive` ternary in the nav link className:

- Active: replace `bg-amber-400/10 text-amber-400` with inline style `{ backgroundColor: "var(--brass-muted)", color: "var(--brass)" }`
- Inactive: use inline style `{ color: "var(--text-secondary)" }` with Tailwind hover classes `hover:bg-[var(--bg-tertiary)] hover:text-[var(--text-primary)]`

Keep all non-color Tailwind classes (flex, gap, px, py, rounded-lg, text-sm, font-medium, transition-colors, etc.) as-is.

- `text-zinc-100` -> `style={{ color: "var(--text-primary)" }}`
- `text-zinc-400` -> `style={{ color: "var(--text-secondary)" }}`

**admin-layout-gate.tsx:**

- `text-zinc-400` -> `style={{ color: "var(--text-secondary)" }}`
