<task type="auto">
  <name>Task 1: Unify CSS variable system and fix contrast</name>
  <files>apps/web/app/globals.css</files>
  <action>
In `globals.css`, consolidate the CSS variable system:

1. **Remove the entire `.dark { ... }` block** (lines 132-164). It is an exact duplicate of `:root`. The `<html className="dark">` in layout.tsx means `:root` values already apply everywhere.

2. **Unify the :root block**: Keep all variables in `:root` but organize them into clear sections with comments:
   - `/* === shadcn semantic colors === */` (the oklch variables: --background through --sidebar-ring)
   - `/* === Ammo Exchange tokens === */` (the custom hex variables: --bg-_, --text-_, --brass-_, --green, --red, --amber, --blue, --border-_)

3. **Fix --text-muted contrast**: Change `--text-muted: #55556a` to `--text-muted: #8585a0`. This gives approximately 5.2:1 contrast ratio against #0a0a0f (--bg-primary), exceeding WCAG AA 4.5:1. Also check --muted-foreground (oklch 0.48) -- change to `oklch(0.63 0.01 280)` to bring it to ~4.5:1 against the background oklch(0.13). These two variables serve similar purposes in different systems so both need upgrading.

4. **Remove the @custom-variant dark line** (line 5: `@custom-variant dark (&:is(.dark *));`). Since we are dark-only, the dark: variant is unnecessary -- all styles are the base styles. However, keep this line ONLY if shadcn components use `dark:` prefixes internally. Check by searching for `dark:` in components/ui -- if found, keep the line. If not found, remove it.

5. **Clean up the @layer base block**: The body rule has both Tailwind utilities (`@apply bg-background text-foreground`) AND inline overrides (`background-color: var(--bg-primary); color: var(--text-primary)`). The inline overrides win, making the @apply line dead code. Remove the `@apply bg-background text-foreground;` line since the explicit var() values are what actually render.

Do NOT change the `@theme inline` block -- it correctly bridges CSS variables to Tailwind utility classes.
</action>
