  <files>

  <action>
    Rewrite the admin orders API route to support:

    1. **Remove PENDING-only filter**: Currently hardcoded to `status: "PENDING"`. Change to accept an optional `status` query param. If not provided, return all statuses. If provided, validate it's a valid OrderStatus value (PENDING, PROCESSING, COMPLETED, FAILED, CANCELLED).

    2. **Text search** via `search` query param: If provided, add a Prisma `OR` condition searching:
       - `id: { contains: search, mode: 'insensitive' }`
       - `walletAddress: { contains: search, mode: 'insensitive' }`
       - `txHash: { contains: search, mode: 'insensitive' }`
       - `onChainOrderId: { contains: search, mode: 'insensitive' }`

    3. **Caliber filter** via `caliber` query param: If provided, validate it's a valid Prisma Caliber enum value (NINE_MM, FIVE_FIVE_SIX, TWO_TWO_LR, THREE_OH_EIGHT). Add `caliber: caliberParam` to the where clause. Use `CALIBER_TO_PRISMA` from shared to accept user-facing caliber names (9MM, 556, 22LR, 308) and convert.

    4. **Pagination** via `page` (default 1) and `limit` (default 20, max 100) query params:
       - Compute `skip = (page - 1) * limit`
       - Use `prisma.order.findMany({ where, skip, take: limit, orderBy: { createdAt: "desc" }, include: { shippingAddress: true, user: { select: { kycStatus: true } } } })`
       - Also run `prisma.order.count({ where })` in parallel to get total
       - Return `{ orders: mappedOrders, total, page, limit, totalPages: Math.ceil(total / limit) }`

    5. Keep the existing `type` filter param as-is. All filters combine with AND logic.

    6. Ensure the mapping still converts caliber via `PRISMA_TO_CALIBER` and dates to ISO strings.

    Import `CALIBER_TO_PRISMA` from `@ammo-exchange/shared` (already available -- it maps e.g. "9MM" -> "NINE_MM").
  </action>

  </verify>
