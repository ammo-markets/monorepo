  <action>
    1. Create `use-refund-mint.ts` following the exact same pattern as `use-finalize-mint.ts`:
       - Single `useWriteContract` + `useWaitForTransactionReceipt` pattern
       - Takes `caliber: Caliber` param, resolves market address from `CONTRACT_ADDRESSES.fuji.calibers[caliber].market`
       - Exposes `refundMint(orderId: bigint)` calling `CaliberMarketAbi` function `refundMint` with args `[orderId]`
       - Returns `{ refundMint, hash, error, isPending, isConfirming, isConfirmed, reset }`

    2. Create `use-cancel-redeem.ts` following the same pattern:
       - Exposes `cancelRedeem(orderId: bigint)` calling `CaliberMarketAbi` function `cancelRedeem` with args `[orderId]`
       - Same return shape as above

    3. Create `reject-mint-dialog.tsx` using the shadcn AlertDialog component (NOT a custom modal like the finalize dialogs use):
       - Import AlertDialog, AlertDialogContent, AlertDialogHeader, AlertDialogTitle, AlertDialogDescription, AlertDialogFooter, AlertDialogCancel, AlertDialogAction from `@/components/ui/alert-dialog`
       - Props: `{ order: AdminMintOrder, open: boolean, onOpenChange: (open: boolean) => void, onRejected: (orderId: string) => void }`
       - Import AdminMintOrder type from `./finalize-mint-dialog`
       - **Required reason field**: Add a `<textarea>` for reason (minimum 1 char, trimmed). Show validation error if empty on submit attempt. Use state for reason and reasonError.
       - On confirm: call `useRefundMint` hook with the order's caliber, invoke `refundMint(BigInt(order.onChainOrderId!))`.
       - On `isConfirmed`: invalidate `queryKey: ["admin"]` queries, call `onRejected(order.id)`, close dialog, reset hook + clear reason.
       - On `error`: show `toast.error(parseContractError(error))`.
       - AlertDialogAction should use `variant="destructive"` (red warning tone, consistent with Phase 20 wallet disconnect)
       - Disable the action button while `isPending || isConfirming`
       - Warning text: "This will refund the user's USDC. This action cannot be undone."
       - AlertDialogAction text: "Reject Order" (or "Submitting..."/"Confirming..." during tx)

    4. Create `cancel-redeem-dialog.tsx` with the same AlertDialog pattern:
       - Props: `{ order: AdminRedeemOrder, open: boolean, onOpenChange: (open: boolean) => void, onCancelled: (orderId: string) => void }`
       - Import AdminRedeemOrder type from `./finalize-redeem-dialog`
       - Uses `useCancelRedeem` hook
       - Warning text: "This will return the user's tokens. This action cannot be undone."
       - AlertDialogAction text: "Cancel Order" (or "Submitting..."/"Confirming..." during tx)

    5. Add exports to `features/admin/index.ts` for both new dialog components.

    NOTE: The AlertDialogAction button text needs to change during tx states. Since AlertDialogAction wraps a Button, you'll need to prevent the default dialog close behavior. Use `onClick` with `e.preventDefault()` to keep the dialog open during the async transaction flow, and manually close via `onOpenChange(false)` on success.
