Replace the stats computation:

2. From `/api/stats` response, extract:
   - `uniqueHolders`: Sum of all `userCount` values across calibers (or take the max if they represent total unique users). The API returns per-caliber rows, so sum the `userCount` field. If no data, show "--".
   - Keep TVL and Rounds Tokenized from `useMarketData` (real-time from chain is better than DB snapshot).
   - For 24h volume: the current /api/stats endpoint doesn't return 24h volume data (it only has totalMinted/totalRedeemed/netSupply/userCount). Show "Coming Soon" instead of "--" for volume24h, or compute an approximation from totalMinted + totalRedeemed. Actually, since we can't get real 24h volume, show the total protocol volume (totalMinted + totalRedeemed summed across calibers) labeled as "Total Volume" instead of "24h Volume".

3. Update the stats object:
   ```typescript
   const stats = useMemo(() => {
     // TVL and rounds from market data (real-time)
     const totalRounds = calibers.reduce((sum, c) => sum + c.totalSupply, 0);
     const tvl = calibers.reduce((sum, c) => sum + c.totalSupply * c.pricePerRound, 0);

     // Unique holders from /api/stats
     const holders = protocolStats?.stats?.reduce((sum, s) => sum + s.userCount, 0) ?? 0;

     // Total volume from /api/stats (totalMinted across all calibers)
     const totalMinted = protocolStats?.stats?.reduce((sum, s) => sum + Number(s.totalMinted), 0) ?? 0;

     return {
       tvl: calibers.length > 0 ? formatCompact(tvl) : "--",
       roundsTokenized: calibers.length > 0 ? totalRounds.toLocaleString("en-US") : "--",
       uniqueHolders: holders > 0 ? holders.toLocaleString("en-US") : "--",
       totalVolume: totalMinted > 0 ? totalMinted.toLocaleString("en-US") : "--",
     };
   }, [calibers, protocolStats]);
   ```
