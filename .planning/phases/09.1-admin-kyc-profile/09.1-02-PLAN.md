  <action>
    Rewrite `apps/web/app/admin/layout.tsx` to be an async server component:

    1. Import:
       - `notFound` from "next/navigation"
       - `getSession` from "@/lib/auth" (NOT requireSession — that throws Response which doesn't work in server components)
       - `publicClient` from "@/lib/viem"
       - `AmmoManagerAbi` from "@ammo-exchange/contracts/abis"
       - `CONTRACT_ADDRESSES` from "@ammo-exchange/shared"
       - `AdminLayoutGate` from "@/features/admin/admin-layout-gate"
       - `AdminSidebar` from "@/features/admin/admin-sidebar"

    2. Make the layout async:
       ```typescript
       export default async function AdminLayout({
         children,
       }: {
         children: React.ReactNode;
       }) {
         // Server-side auth check
         const session = await getSession();

         if (!session.siwe) {
           notFound();
         }

         // Server-side keeper check
         const isKeeper = await publicClient.readContract({
           address: CONTRACT_ADDRESSES.fuji.manager,
           abi: AmmoManagerAbi,
           functionName: "isKeeper",
           args: [session.siwe.address as `0x${string}`],
         });

         if (!isKeeper) {
           notFound();
         }

         return (
           <AdminLayoutGate>
             <div className="flex h-screen">
               <AdminSidebar />
               <main className="flex-1 overflow-y-auto p-8">
                 <div className="mx-auto max-w-5xl">{children}</div>
               </main>
             </div>
           </AdminLayoutGate>
         );
       }
       ```

    3. Why `notFound()` instead of redirect:
       - Non-keepers should see 404, not be redirected (avoids revealing the admin route exists)
       - `notFound()` is the correct server component approach (throws NEXT_NOT_FOUND internally)

    4. Why keep `AdminLayoutGate`:
       - It provides a nice loading state during client-side hydration
       - It's a safety net if the server-side check somehow passes but the client-side check fails
       - Removing it would flash the admin UI briefly during hydration in edge cases

  </action>
