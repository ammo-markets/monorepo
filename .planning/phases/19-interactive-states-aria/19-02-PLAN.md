  <action>
Systematically audit every file in `apps/web/features/` and `apps/web/app/` for interactive elements that need ARIA attributes.

**Priority 1: Icon-only buttons (A11Y-03)**

Search for buttons that contain ONLY an icon (Lucide icon component) without visible text. Add `aria-label` describing the action.

Examples of what to find and fix:
- Buttons with just `<Copy size={14} />` -> add `aria-label="Copy to clipboard"`
- Buttons with just `<X size={16} />` -> add `aria-label="Close"`
- Buttons with just `<ExternalLink size={14} />` -> add `aria-label="View on explorer"`
- Buttons with just `<ChevronLeft />` / `<ChevronRight />` -> add `aria-label="Previous"` / `aria-label="Next"`
- Buttons with just `<Eye />` or `<EyeOff />` -> add `aria-label="Show password"` / `aria-label="Hide password"`
- Buttons with just `<RefreshCw />` -> add `aria-label="Refresh"`

For buttons that already have `<span className="sr-only">` text (like shadcn sidebar trigger), they are already accessible -- skip them.

**Priority 2: Active navigation states**

In app-nav.tsx, add `aria-current="page"` to the active nav link:
```tsx
// When isActive is true
aria-current={isActive ? "page" : undefined}
```

**Priority 3: Tab components**

In trade-tabs.tsx, ensure tab buttons have:
- `role="tab"` (if not already using shadcn Tabs component)
- `aria-selected={isActive}`
- The tab container has `role="tablist"`

In time-range-selector.tsx (which acts like tabs), add similar:
- `role="tab"` on each time range button
- `aria-selected={isActive}` on each
- `role="tablist"` on the container

**Priority 4: Form inputs and buttons with context**

In mint-flow.tsx, redeem-flow.tsx, swap-widget.tsx -- ensure:
- Number inputs have `aria-label` if no visible `<label>` is associated (e.g., "Amount to mint", "Amount to redeem")
- Step indicators have `aria-label` describing which step (e.g., "Step 1 of 4: Select caliber")
- Action buttons with icons + text are fine (text provides accessibility)

**Priority 5: Links that open external sites**

For external links (snowtrace, etc.), consider adding:
- `aria-label` that includes "opens in new tab" context, OR
- Ensure `target="_blank"` links have `rel="noopener noreferrer"` (security + accessibility)

**Do NOT add aria-labels to:**
- Buttons that already have visible text (text provides accessibility)
- Buttons with existing `<span className="sr-only">` text
- Links with visible text content
- Decorative icons that are inside labeled buttons
