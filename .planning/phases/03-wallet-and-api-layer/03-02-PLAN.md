    **2. Create `apps/web/lib/serialize.ts`:**

    A utility function to safely serialize objects containing BigInt values for JSON responses. Prisma returns `BigInt` for the `amount` and `lastBlock` fields, and viem returns `bigint` for uint256 values. `JSON.stringify(42n)` throws `TypeError: Do not know how to serialize a BigInt`.

    ```typescript
    export function serializeBigInts<T>(obj: T): T {
      return JSON.parse(
        JSON.stringify(obj, (_, value) =>
          typeof value === "bigint" ? value.toString() : value
        )
      );
    }
    ```

    This handles nested objects and arrays. All API routes that return Prisma records or viem results will use this before `Response.json()`.

    **1. `apps/web/app/api/orders/route.ts` (API-01: GET /api/orders):**

    Import `NextRequest` from `next/server`, `z` from `zod`, `prisma` from `@ammo-exchange/db`, `PRISMA_TO_CALIBER` from `@ammo-exchange/shared`, and `serializeBigInts` from `@/lib/serialize`.

    Define a zod schema for the wallet query parameter: `z.object({ wallet: z.string().regex(/^0x[a-fA-F0-9]{40}$/) })`.

    In the GET handler:
    - Parse `request.nextUrl.searchParams.get("wallet")` with the zod schema
    - Return 400 with `{ error: "Invalid wallet address" }` if validation fails
    - Query `prisma.order.findMany({ where: { walletAddress: parsed.data.wallet.toLowerCase() }, orderBy: { createdAt: "desc" }, include: { shippingAddress: true } })`
    - CRITICAL: `.toLowerCase()` the wallet address before querying (prior decision: all wallet addresses stored lowercase)
    - Map orders to convert Prisma caliber enum to shared Caliber type using `PRISMA_TO_CALIBER`
    - Use `serializeBigInts` on the response to convert BigInt `amount` fields to strings
    - Return `Response.json({ orders: serializeBigInts(mappedOrders) })`

    Also support optional `type` query param to filter by MINT or REDEEM: `z.object({ wallet: ..., type: z.enum(["MINT", "REDEEM"]).optional() })`. Add `type` to the Prisma `where` clause if provided.

    **5. `apps/web/app/api/redeem/shipping/route.ts` (API-04: POST /api/redeem/shipping):**

    Import `NextRequest` from `next/server`, `z` from `zod`, `prisma` from `@ammo-exchange/db`, and `RESTRICTED_STATES` from `@ammo-exchange/shared`.

    Define zod schema:
    ```typescript
    const shippingSchema = z.object({
      orderId: z.string().min(1),
      name: z.string().min(1).max(100),
      line1: z.string().min(1).max(200),
      line2: z.string().max(200).optional(),
      city: z.string().min(1).max(100),
      state: z.string().length(2).refine(
        (s) => !RESTRICTED_STATES.includes(s as (typeof RESTRICTED_STATES)[number]),
        { message: "Shipping to this state is restricted" }
      ),
      zip: z.string().regex(/^\d{5}(-\d{4})?$/),
    });
    ```

    In the POST handler:

    - Upsert shipping address: `prisma.shippingAddress.upsert({ where: { orderId }, create: { orderId, ...address }, update: address })`.
    - Return 201 with `{ shipping }`.
