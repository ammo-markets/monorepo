Create `apps/worker/src/indexer.ts` -- the core indexer logic.

**Imports:**

- `client` from `./lib/client`
- `BATCH_SIZE`, `MARKET_ADDRESSES`, `CURSOR_KEY`, `CHAIN_ID` from `./lib/constants`
- `type { EventMeta }` from `./lib/constants`
- `getCursor`, `upsertCursor` from `./lib/cursor`
- `handleMintStarted`, `handleMintFinalized` from `./handlers/mint`
- `type { MintStartedArgs, MintFinalizedArgs }` from `./handlers/mint`
- `handleRedeemRequested`, `handleRedeemFinalized` from `./handlers/redeem`
- `type { RedeemRequestedArgs, RedeemFinalizedArgs }` from `./handlers/redeem`
- `prisma` from `@ammo-exchange/db`
- `CaliberMarketAbi` from `@ammo-exchange/contracts`

**fetchEvents(fromBlock: bigint, toBlock: bigint):**

- Fetch all 4 event types in parallel using `Promise.all` with `client.getContractEvents`:
  - Each call uses `abi: CaliberMarketAbi`, `address: MARKET_ADDRESSES`, `fromBlock`, `toBlock`, `strict: true`
  - Event names: `"MintStarted"`, `"MintFinalized"`, `"RedeemRequested"`, `"RedeemFinalized"`
- Return object `{ mintStarted, mintFinalized, redeemRequested, redeemFinalized }` with the 4 arrays

**processAndCommit(events, lastBlock: bigint):**

- Combine all 4 event arrays into a single array
- Sort by `blockNumber` ascending, then `logIndex` ascending (ensures MintStarted processes before MintFinalized within same block)
- Open `prisma.$transaction(async (tx) => { ... }, { timeout: 30_000 })`:
  - For each event in sorted order, switch on `event.eventName`:
    - `"MintStarted"`: call `handleMintStarted(tx, event.args as MintStartedArgs, meta)` where meta = `{ address: event.address, transactionHash: event.transactionHash, blockNumber: event.blockNumber, logIndex: event.logIndex }`
    - `"MintFinalized"`: call `handleMintFinalized(tx, event.args as MintFinalizedArgs, meta)`
    - `"RedeemRequested"`: call `handleRedeemRequested(tx, event.args as RedeemRequestedArgs, meta)`
    - `"RedeemFinalized"`: call `handleRedeemFinalized(tx, event.args as RedeemFinalizedArgs, meta)`
  - After all events processed: call `upsertCursor(tx, lastBlock)` to atomically advance the checkpoint
- Log the number of events processed and the block range

**pollOnce():**

- Get current block: `const currentBlock = await client.getBlockNumber()`
- Read cursor inside a quick transaction or directly: `const cursor = await prisma.blockCursor.findUnique({ where: { contractAddress: CURSOR_KEY } })`
- Compute `fromBlock = (cursor?.lastBlock ?? 0n) + 1n`
- If `fromBlock > currentBlock`, log "caught up" and return (no work)
- Process in batches of BATCH_SIZE:
  - `let batchStart = fromBlock`
  - While `batchStart <= currentBlock`:
    - `const batchEnd = batchStart + BATCH_SIZE - 1n > currentBlock ? currentBlock : batchStart + BATCH_SIZE - 1n`
    - `const events = await fetchEvents(batchStart, batchEnd)`
    - `const totalEvents = events.mintStarted.length + events.mintFinalized.length + events.redeemRequested.length + events.redeemFinalized.length`
    - If totalEvents > 0: call `processAndCommit(events, batchEnd)` and log batch info
    - If totalEvents === 0: just update cursor to batchEnd (so we don't re-scan empty ranges). Do this via a simple `prisma.blockCursor.upsert(...)` outside a full transaction since there are no events to process atomically.
    - `batchStart = batchEnd + 1n`
- Log polling summary (blocks scanned, events found)

**backfill():**

- Same logic as pollOnce() but with a log prefix of "[backfill]" instead of "[poll]"
- Actually, `pollOnce` IS the backfill -- it processes all blocks from cursor to current head in batches. So export `backfill = pollOnce` as an alias, OR just export `pollOnce` and call it for both backfill and polling. The distinction is cosmetic (logging).
- Simplest: just export `pollOnce`. The entry point will call it once for backfill (logging "Backfilling...") and then repeatedly in the polling loop.

Export: `pollOnce`

Imports:

**main() async function:**

1. Log startup: `[worker] Starting Ammo Exchange event indexer...`
2. Log chain info: `[worker] Chain: ${client.chain.name} (${client.chain.id})`
3. Get and log current block: `const blockNumber = await client.getBlockNumber()` -> `[worker] Current block: ${blockNumber}`
4. Run initial backfill: `console.log("[worker] Running initial backfill...")` then `await pollOnce()` then `console.log("[worker] Backfill complete.")`
5. Start polling loop:
   ```
   let isProcessing = false;
   const intervalId = setInterval(async () => {
     if (isProcessing) return;
     isProcessing = true;
     try {
       await pollOnce();
     } catch (error) {
       console.error("[worker] Polling error (will retry):", error);
     } finally {
       isProcessing = false;
     }
   }, POLL_INTERVAL_MS);
   ```
6. Log: `[worker] Polling every ${POLL_INTERVAL_MS / 1000}s for new events...`
7. Graceful shutdown handlers:
   ```
   const shutdown = () => {
     console.log("[worker] Shutting down...");
     clearInterval(intervalId);
     process.exit(0);
   };
   process.on("SIGTERM", shutdown);
   process.on("SIGINT", shutdown);
   ```

**Error boundary:**

- Call `main().catch((err) => { console.error("[worker] Fatal error:", err); process.exit(1); })`
- This catches startup/backfill errors as fatal (process exits)
- Polling errors are caught inside the interval and logged but do NOT kill the process
