Create two event handler modules in `apps/worker/src/handlers/`:

**mint.ts** -- MintStarted and MintFinalized handlers:

Import `type { PrismaTx }` from `../lib/cursor`, `addressToCaliber` from `../lib/constants`, `CALIBER_TO_PRISMA` from `@ammo-exchange/shared`, `type { Caliber as PrismaCaliber }` from `@ammo-exchange/db`, `CHAIN_ID` from `../lib/constants`.

Define event arg types inline based on the CaliberMarket ABI events (do NOT import the ABI in handlers -- keep them decoupled from viem log types). Instead, define a clean interface for each:

```typescript
export interface MintStartedArgs {
  orderId: bigint;
  user: `0x${string}`;
  usdcAmount: bigint;
  requestPrice: bigint;
  minTokensOut: bigint;
  deadline: bigint;
}

export interface MintFinalizedArgs {
  orderId: bigint;
  user: `0x${string}`;
  tokenAmount: bigint;
  priceUsed: bigint;
}

export interface EventMeta {
  address: `0x${string}`;
  transactionHash: `0x${string}`;
  blockNumber: bigint;
  logIndex: number;
}
```

Export `handleMintStarted(tx: PrismaTx, args: MintStartedArgs, meta: EventMeta): Promise<void>`:

- Resolve caliber via `addressToCaliber(meta.address)`
- Map to Prisma caliber via `CALIBER_TO_PRISMA[caliber]`
- Lowercase the user address for consistent storage
- Use `tx.order.upsert` with:
  - `where: { txHash: meta.transactionHash }` (idempotent -- prevents duplicates on reprocessing)
  - `create`: type MINT, status PENDING, caliber (Prisma enum), amount = args.usdcAmount, onChainOrderId = `args.orderId.toString()`, walletAddress = user (lowercased), txHash = meta.transactionHash, chainId = CHAIN_ID
  - For user relation: use `connectOrCreate` with `where: { walletAddress: user.toLowerCase() }` and `create: { walletAddress: user.toLowerCase() }`
  - `update: {}` (no-op if already exists -- idempotent)

Export `handleMintFinalized(tx: PrismaTx, args: MintFinalizedArgs, meta: EventMeta): Promise<void>`:

- Resolve caliber and Prisma caliber same as above
- Use `tx.order.updateMany` with:
  - `where: { onChainOrderId: args.orderId.toString(), caliber: prismaCaliber, type: "MINT", status: "PENDING" }`
  - `data: { status: "COMPLETED" }`
- `updateMany` is used instead of `update` because `onChainOrderId` is not a unique field. This finds the matching PENDING mint order for this caliber and marks it COMPLETED.
