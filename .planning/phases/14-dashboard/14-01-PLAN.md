**balance-cards.tsx** -- `BalanceCards` component:

- Accept props: `balances: Record<Caliber, bigint | undefined>`, `usdc: bigint | undefined`, `marketData: MarketCaliberFromAPI[]`, `isLoading: boolean`
- Display a responsive grid (2 cols on mobile, 4 cols on lg) of cards, one per caliber (9MM, 556, 22LR, 308)
- Each card shows: caliber icon (from `@/features/shared/caliber-icons`), caliber symbol, balance in rounds (formatUnits with 18 decimals, Math.floor), USD value (balance \* pricePerRound from marketData), and price per round
- Below the 4 caliber cards, show USDC balance (formatUnits with 6 decimals) in a smaller row
- Total portfolio value summed at top as a header: "$X.XX" with label "Portfolio Value"
- Loading state: shimmer skeleton cards matching layout
- Empty state (all balances zero): show all 4 cards with "0 rounds" and "$0.00"
- Use `caliberIcons` from `@/features/shared/caliber-icons` for icons
- Import types: `import type { Caliber } from "@ammo-exchange/shared"`, `import type { MarketCaliberFromAPI } from "@/lib/types"`

**recent-orders.tsx** -- `RecentOrders` component:

- Accept props: `orders: OrderFromAPI[]`, `isLoading: boolean`
- Show the first 5 orders (slice from already-sorted-desc array)
- Each row shows: order type badge (Mint green / Redeem amber), caliber icon + name, amount in rounds, status badge (Processing blue / Completed green / Failed red), and relative timestamp (e.g., "2 days ago" using simple helper -- compute from Date.now() - Date.parse(createdAt))
- Desktop: compact table rows. Mobile: stacked cards (follow portfolio-dashboard.tsx pattern)
- "View All" link at bottom navigating to `/portfolio`
- Loading state: 3-row shimmer skeleton
- Empty state: "No orders yet. Start by minting some tokens." with link to /mint
- Reuse `StatusBadge` and `TypeBadge` inline (or define locally) matching portfolio-dashboard.tsx patterns
- Import: `import type { OrderFromAPI } from "@/lib/types"`

**quick-actions.tsx** -- `QuickActions` component:

- No props needed (pure navigation component)
- Two prominent buttons side by side in a card:
  - "Mint Tokens" (brass background, links to `/mint`) with a Plus or ArrowUpRight icon from lucide-react
  - "Redeem Tokens" (outline style with border, links to `/redeem`) with an ArrowDownLeft icon from lucide-react
- Responsive: stack vertically on mobile, side by side on sm+
- Use `<Link>` from `next/link` for client-side navigation

**pending-banner.tsx** -- `PendingBanner` component:

- Accept props: `pendingCount: number`
- If `pendingCount === 0`, render nothing (return null)
- If `pendingCount > 0`, render an amber/warning banner at the top of the dashboard:
  - Icon: AlertTriangle from lucide-react
  - Text: "You have {N} pending order(s) awaiting processing"
  - "View Orders" link navigating to `/portfolio`
- Style: amber background tint, amber text, rounded-xl, matches existing color variables

**index.ts** -- barrel file:

- Export all 4 components: `export { BalanceCards } from "./balance-cards"`, etc.

All components must be `"use client"` (they use hooks or event handlers).
Exception: `QuickActions` can be server component if using only `<Link>` -- but mark `"use client"` for consistency since the page is client-rendered.
