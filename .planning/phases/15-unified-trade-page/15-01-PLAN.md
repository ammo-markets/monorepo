**caliber-info-panel.tsx** -- A "use client" component that:
- Accepts `selectedCaliber: Caliber | null`, `onSelectCaliber: (cal: Caliber) => void`, and `marketData: MarketCaliberFromAPI[]` as props
- Renders a horizontal row (desktop) / scrollable row (mobile) of 4 caliber cards (9MM, 556, 22LR, 308)
- Each card shows: caliber icon (from `caliberIcons`), symbol, name, current price per round from marketData, and a compact specs line showing grain weight + case type + min order (e.g. "115gr | Brass | Min 50 rds") pulled from `CALIBER_SPECS`
- Selected card has brass border/highlight (same pattern as mint-flow's StepSelectCaliber)
- Cards use `grid grid-cols-2 sm:grid-cols-4 gap-3` layout
- When no caliber is selected, show a subtle prompt "Select a caliber to begin"
- Import `CALIBER_SPECS` from `@ammo-exchange/shared`, `caliberIcons` from `@/features/shared/caliber-icons`, types from `@/lib/types`
- Follow existing codebase CSS variable patterns (var(--bg-secondary), var(--brass), var(--border-default), etc.) -- no Tailwind color classes

**trade-tabs.tsx** -- A "use client" component that:
- Accepts `selectedCaliber: Caliber | null` and `activeTab: "mint" | "redeem" | "swap"` and `onTabChange` as props
- Renders 3 tab buttons: "Mint" (with Plus icon), "Redeem" (with ArrowDownToLine icon), "Swap" (with ArrowDownUp icon) from lucide-react
- Active tab uses brass background/text, inactive tabs use muted styling (same pattern as SwapWidget's tab bar)
- Tab content area:
  - **Mint tab**: Renders `<MintFlow />` from `@/features/mint` wrapped in Suspense. The MintFlow already handles its own caliber selection, amount entry, and order summary internally. Pass `?caliber={selectedCaliber}` as a URL search param by wrapping in a div that sets the search param so MintFlow's `useSearchParams` picks it up. Actually, simpler approach: render MintFlow as-is since it has its own caliber selector built in. The CaliberInfoPanel at the top provides context/quick reference, but the flow manages its own state.
  - **Redeem tab**: Renders `<RedeemFlow />` from `@/features/redeem` wrapped in Suspense. Same approach -- RedeemFlow has its own built-in caliber+amount selection.
  - **Swap tab**: Renders `<SwapWidget defaultOpen />` from `@/features/trade` inline (not as a modal)
- When selectedCaliber is set from the CaliberInfoPanel AND the user clicks a tab, pass `?caliber={selectedCaliber}` as URL search params using `useRouter().replace()` so the MintFlow/RedeemFlow can pre-select it via their existing `useSearchParams` logic
- Tabs should be visually distinct pill-style buttons in a row, centered above the content
