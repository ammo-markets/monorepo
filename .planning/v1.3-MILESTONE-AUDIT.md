## Tech Debt by Phase

### Phase 15: Unified Trade Page

**1. Duplicate Caliber Selection (user-reported)**
- **Files:** `apps/web/features/trade/caliber-info-panel.tsx` + `apps/web/features/mint/mint-flow.tsx`
- **Issue:** CaliberInfoPanel renders 4 caliber cards at top of Trade page. MintFlow Step 0 renders "Choose Your Caliber" with same 4 cards below. User selects caliber twice.
- **Root cause:** CaliberInfoPanel syncs selection to `?caliber=` URL param, but MintFlow doesn't read this param to skip step 0.
- **Fix options:**
  - A) MintFlow detects URL param and starts at step 1 if valid caliber provided
  - B) Remove caliber selection from CaliberInfoPanel (make it display-only, no click)
  - C) Pass `skipCaliberSelection` prop to MintFlow when inside Trade page

### Phase 12: Database Schema & Stats Worker

**2. Orphaned /api/stats endpoint**
- **File:** `apps/web/app/api/stats/route.ts`
- **Issue:** Worker computes per-caliber stats every 15 minutes into ProtocolStats table, but ProtocolStats UI component uses `useMarketData` instead. Landing page shows "--" for unique holders and 24h volume.
- **Fix:** Connect ProtocolStats component to `/api/stats` OR remove endpoint

**3. Orphaned /api/user/preferences endpoint**
- **File:** `apps/web/app/api/user/preferences/route.ts`
- **Issue:** Created and authenticated but no UI component consumes it in v1.3
- **Fix:** Document as "ready for future use" or create preferences UI

**4. DB-03 partial scope**
- **Issue:** UserPreference only stores favoriteCalibers. Requirement mentions "display settings and notification preferences" but plan deliberately scoped to favorites only.
- **Fix:** Track as future enhancement

### Total: 4 items across 2 phases
